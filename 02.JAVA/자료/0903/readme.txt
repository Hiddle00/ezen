1) DI 개념 (Dependency Injection : 의존성 주입)
   결합도 : 모듈과 모듈 사이의 의존 정도 (결합도는 낮을 수록 좋다.)
              클래스 A가 클래스 B에 의존하고 있을 때 A와 B 사이의 결합도가 낮다는
              것은 클래스 B가 변경되었을 때 클래스 A에는 영향을 주지 않는다는 것을 
              의미합니다. B의 코드를 수정해도 A는 영향을 받지 않습니다. 따라서 나도 
              모르게 다른 클래스를 고장낼 확률이 낮아집니다.
   응집도 : 모듈 내부의 기능적인 집중 정도 (응집도는 높을 수록 좋다.)
              같은 목적을 가진 데이터와 메서드가 하나의 클래스에 모여있을수록 응집도가 
              높다고 할 수 있습니다. 클래스의 책임이 명확해지므로 코드를 분석하고 
              이해하기 쉬워집니다. 재활용이 쉬워지므로 코드 중복을 줄이는 데 큰 도움이 됩니다.
